# Venture OS Pilot Onboarding

## Pilot Goal

Onboard internal GTCX product repos and warm external clients into a repeatable
venture-growth operating system within five business days.

## Day 0: Intake

Collect:

1. Company/product name.
2. Founder/team contacts.
3. Geography and markets.
4. Sector and customer type.
5. Current stage.
6. Capital goal.
7. Revenue/pilot goal.
8. Partnership goal.
9. Visibility goal.
10. Existing deck, one-pager, website, demo, financial model.
11. Current investors/partners/customers.
12. Sensitive claims and constraints.
13. Email/calendar/document systems.

## Day 1: Profile And Positioning

Outputs:

- Client profile.
- Product/offering map.
- Buyer/funder/partner ICPs.
- Core narrative.
- Proof-point inventory.
- Risk/claims constraints.

## Day 2: Pipeline Build

Outputs:

- First 25-50 candidate opportunities.
- Segmented by capital, revenue, partnership, visibility, investor, program.
- P1/P2 prioritization.
- Owners and next actions.

## Day 3: Collateral And Outreach Setup

Outputs:

- Collateral inventory.
- Missing asset list.
- Outreach templates customized.
- Warm intro list.
- First draft assets.

## Day 4: Operating System Setup

Outputs:

- ClickUp/client board.
- Gmail/calendar labels or manual workflow.
- Drive folder structure.
- Approval workflow.
- Daily/weekly brief format.

## Day 5: Founder Review

Agenda:

1. Review client profile and positioning.
2. Review top opportunities.
3. Approve first outreach/collateral priorities.
4. Confirm weekly cadence.
5. Assign founder approvals and relationship owners.

## Pilot Success Criteria

Within 30 days:

| Metric | Target |
| ------ | ------ |
| Qualified opportunities | 20+ |
| P1 opportunities | 5+ |
| Outreach drafts | 10+ |
| Approved external sends | 5+ |
| Meetings booked | 2+ |
| Collateral assets drafted | 2+ |
| Visibility opportunities pursued | 3+ |
| Founder weekly briefs delivered | 4 |
| External actions without approval | 0 |
